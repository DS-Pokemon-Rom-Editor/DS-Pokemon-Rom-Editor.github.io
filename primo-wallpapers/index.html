How does this work?

Algorithm Explanation

The password system encodes your Trainer ID and desired wallpaper into 4 Easy Chat words using the following steps:

  1. Build the 4-byte payload:
    • Byte 0 = 0x60 | wallpaper (magic nibble 6 + wallpaper 0-7)
    • Byte 1 = (trainerID >> 8) XOR byte0
    • Byte 2 = (trainerID & 0xFF) XOR byte0
    • Byte 3 = ((byte2 XOR byte0) × (byte0 + (byte1 XOR byte0))) & 0xFF (checksum)
  2. Rotate left 3 bytes by (byte3 & 0x0F) bits
  3. XOR bytes 0,1,2 with transform(byte3) where transform duplicates high nibble: 0xAB → 0xAA
  4. Rotate left 4 bytes by 5 bits
  5. Decode deltas to word indices: Convert the 4 bytes back to Easy Chat word indices using delta encoding with wraparound
  6. Look up words: Convert indices to the actual Easy Chat words from the password bank (351 words)
